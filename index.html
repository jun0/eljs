eelll/JS とは

eelll/JS は、 伝統ある T-Code 練習プログラム DOGGG (どっぐぐぐ), EELLL (うなぎぎぎ、tc2 に付属) の JavaScript 版です。 TUT-Code にも対応しています。 MSIE6 などで動作します。 要 JavaScript。Cookie 推奨。 GNU GPL ライセンス。

コード検索専用の 漢索/JS も付属しています。

特徴

  • 近代的な Web ブラウザの多くでおそらく動作します。 Windows, MacOS, Linux などプラットフォームを選びません。
  • T-Code, TUT-Code, G-Code など、 代表的な無想式直接入力方式の比較的多くに対応しています。
    内部に漢字コード逆変換表を持っているため、 直接入力のための環境を用意しなくても練習できます。 他の入力方式にも、 ユーザ定義機能によってある程度は対応できます。
  • 複数の練習テキストに対応しています。 デフォルトでは、DOGGG や EELLL でおなじみの T-Code の標準テキスト EELLLTXT や、 TUT-Code の標準的なテキスト kanji_a1.txt などを用意しています。
  • 橋田表風の打鍵図で、コード (文字の打ち方) を表示できます。

動作環境

  • JavaScript と DOM にそれなりに対応しているブラウザ。
    Internet Explorer 6.0, Netscape 7.1, Opera 7.54u2, FireFox 1.5.0.1 (以上、WindowsXP)、 Internet Explorer 5.0 (MacOS 8.5) で動作確認しています。
    Mozilla 1.8a3 (WindowsXP) では動作しないという報告がありました。(eelll/JS v0.1)
  • クッキーを有効にしておくと、 練習の設定 (入力方式やレッスンなど) をブラウザが記憶してくれますので、 幸せになれるでしょう。
  • なるべく高速なマシンと、なるべく高速なネットワーク環境を、強く推奨。

使い方

  1. eelll/JS のページに進む

    ページの読みこみに少し時間がかかりますが、じっとがまんしてください。 IME や 漢直Win は OFF にしておきます。

    諸般の事情から、オフラインで練習したい人がいるかもしれません。 …が、 eelll/JS 0.2 では、 打鍵図の表示に画像ファイルを使用しているので、 ページの読みこみ完了後もオンラインのままにしておく方がよいです。 (画像を使わない英字表記ヒントなら、オフラインにしても構いません)

    打鍵図を表示してほしいけれど、 どうしてもオフラインで練習したいという場合は、 [設定(C)] - [デバッグ(D)] - [IMG Preload] を選択し、 [preload] ボタンをクリックして、 すべての画像を読みこんでからオフラインにするようにすると、 うまくいく …かもしれません。

  2. 設定する

    画面上半分にある [設定(C)] セクションで、[入力方式(I)] と [練習テキスト(T)] を選び、 [OK] ボタンをクリックします。

    キーボードにこだわりのある人は、 [キーボード(K)] の項目も適当に選んでください。 QWERTY 日本語 (デフォルト)、QWERTY 英語、Dvorak から選べます。 これ以外のキーボード配列には残念ながら対応していません。 ごめんなさい。

    ここでの設定は、キーコードは関係なく、 そのキーを押した時にどの文字が実際に入力されるかで決まります。 たとえば、 日本語キーボードを窓使いの憂鬱で英語配列化している場合は、 [QWERTY 英語] を選びます。

    なお、T-Code, TUT-Code, G-Code に関しては、 [QWERTY 日本語] と [QWERTY 英語] の違いはありません。

    [入力方式(I)] の選択岐にない方式も、 ユーザ定義を行うことによって、 (eelll/JS が対応できる範囲内で) 利用することができます。

    ユーザ定義を行う場合の、設定の手順は次のとおりです。

    1. [入力方式(I)] から [ユーザ定義] を選択します。 すると、定義を書き込むためのテキストエリアが現れます。
    2. テキストエリアに、 文字 = コード の書式で一行ずつ定義を書いてください。 たとえば、次のように書きます。 
      あ=yf
い=hd
う=yd
愛=fh
言=gl
    3. その他の項目 ([練習テキスト(T)] など) を適宜選択してから、 [OK(O)] または [適用] ボタンをクリックします。
    4. あとは、通常どおりに練習を行います。

    なお、 [ユーザ定義] の定義内容は、クッキーに保存されません。 消えてしまうと困るものは、 手元のファイルに保存しておいてください。

    コードを定義する際は、次の点に注意してください。

    1. 「#」や「;」で始まる行はコメントとして読みとばされます。 が、行の途中の「#」や「;」は、コード定義の一部 とみなされます。
      → #=sharp のような定義はできません (ぁ=# は可能です)。
    2. 文字 の部分は、 1 文字 でなければなりません。
      → った=tta や きゃ=kya のような定義はできません。
    3. コード の部分は、 1 文字以上の (印字可能な) 文字列でなければなりません。
      → F1 などのファンクションキー、 変換・無変換 キー、 Shift+0 などは、 コードとして使うことはできません。 また、 Shift+Space と Space、 ＼ と ￥ などは、 それぞれ同一視されます。
    4. 一つの 文字 に対して、 コード は高々一つでなければなりません。
      → し=si と し=shi の両方を書いた場合は、 あとに書いた方だけが有効です。
    5. コード は、 QWERTY 日本語配列のキーで表記します。
    6. 空行は適宜挿入してもかまいせんが、 不必要な空白やタブ文字は書かないようにしてください (コード定義の一部とみなされる可能性があります)。 特に = の両側に空白は入れないでください。
    7. 書式に合わない行は、(コメント行と同様に) 警告なしに読みとばされます。

    以下は暫定的な仕様です。

    • #include T のように書くと、その場所に、 T-Code の全定義を書き込んだことと同等になります。 既存の入力方式をカスタマイズ (定義の追加・上書き) する場合に便利です。 T のほかに、 TUT, G, Try, TT, phoenix を指定できます。 大文字・小文字の違いは区別されますので、注意してください。
    • 澤~沢 のように書くと、『沢』と同じコードを、 『澤』にも割り当てます (『沢』にはすでにコードが割り当てられているとします)。 『~』(tilde) の右は、二文字以上で定義することができます。

    TUT-Code で、 タッチ17 を実現するには、[ユーザ定義] に以下の内容を指定します。 ただし、半濁音のもともとの 4 ストローク定義は使えなくなります。

    同様の方法で、Touch16+ (→ 森田祐二氏の TUT-Code - Touch16+ (http://www.gar.sakura.ne.jp/tut-code/touch16+.html)) などを実装することもできます。 

    #include TUT

ぱ=t;k
ぴ=t;i
ぷ=t;u
ぺ=t;h
ぽ=t;j
づ=d;u
集=t:
長=d:
パ~ぱ
ピ~ぴ
プ~ぷ
ペ~ぺ
ポ~ぽ
ヅ~づ

    次のように [ユーザ定義] を行うと、 T-Code の部首合成をエミュレートできる …かもしれません。 ただし、一つの文字に一つの定義しか書けませんし、 文字ヘルプも見づらいですし、 そもそも部首合成定義を全部書くのは大変なので、 こんなこともできるぐらいに思ってください。 

    #include T

◆=jf
林~◆木木
鍵~◆金建
冬~◆終糸

    [ユーザ定義] の例として、 越川和忠氏による『き』(の JIS X 0208 部分のみ) を エミュレートし (ようとし) た定義例を示します。
    → ki_data.txt (そのまま copy & paste して使えます)

    本来の『き』では、日本語キーボードの 48 鍵とスペースキー、 およびそのシフトケースをフルに使用しますが、 eelll/JS では、 ￥ は ＼ と、 Shift+Space は Space と、 それぞれ同一視され、また、 Shift+0 は扱えないため、 ここでは、その点を妥協した、不完全な定義になっています。 たとえば、本来は、 『？』は「? Shift+Space」、 『支』は「i Shift+0」ですが、 それぞれ ？=? 、 支=i と定義しています。

    [ユーザ定義] でうまく定義できない入力方式も、 最後の手段として、 [ユーザ定義] の定義内容を空っぽにして設定しておいて、 練習時に IME や 漢直Win で入力する、という方法もあります。 もっとも、練習時は入力キーのエコーバックがないため、 IME では練習しづらいかもしれません。 また、この場合は、文字ヘルプの機能は使えなくなります。

    [練習テキスト] も [ユーザ定義] が可能です。 設定の手順は次のとおりです。

    1. [練習テキスト(T)] から [ユーザ定義] を選択します。 すると、テキストを書き込むためのテキストエリアが現れます。
    2. テキストエリアに、練習したい文章を書きこんでください。 「#」で始まる行はコメントとして読みとばされます。 また、空行を入れると、そこがレッスンの区切りになります。 たとえば、次の例は、一課が二行ずつからなるレッスンを、 三課分定義しています。 
      の、が、のが
て、ての、と、のと、とが

なにをしないでいたいの。
はるになると、たのしい。

うろうろしていて、おろされた。
かっこでくくることができる。
    3. その他の項目 ([入力方式(I)] など) を適宜選択してから、 [OK(O)] または [適用] ボタンをクリックします。
    4. あとは、通常どおりに練習を行います。

    なお、 [ユーザ定義] の定義内容は、クッキーに保存されません。 消えてしまうと困るものは、 手元のファイルに保存しておいてください。

  3. レッスンの準備をする

    [練習(L)] セクションの出力欄に、 『リターンキーを打てば始まります』 と表示されています。 出力欄の下の入力欄 (細長いテキストフィールド) をクリックするなどして、フォーカスを移動してください。

    入力欄の下のヒント欄に、 新しく練習する文字 (もしあれば) の打ち方が表示されています。 レッスンを始める前に、よく見て確認しておきましょう。

    心の準備ができたら、リターンキーを押してください。 レッスンが始まります。

    アクセスキー J で、 入力欄にフォーカスを移すことができます。 (Windows の場合は Alt+J、 MacOS の場合は Ctrl+J。 Opera なら Shift+Esc J)

    練習文は課 (レッスン) に分かれています。 通常は、最初のレッスンから順番に進めていくとよいのですが、 [レッスン(S)] から、目的のレッスンを直接選ぶこともできます。

    [ランダム(R)] ボタンをクリックと、レッスンをランダムに選びます。

    [シャッフル(F)] のチェックをオンにすると、 一つのレッスンの中で、問題文をシャッフルして出題します。

  4. 入力する

    練習テキストの問題文が一行ずつ表示されますので、 T-Code や TUT-Code などで入力してください。

    入力したキーは、画面には表示されません。 まちがいを気にせずに (BackSpace で消したりせずに) どんどん入力しましょう。

    問題文の中の、赤の下線つきの文字は、 前回まちがえた文字です。 同じあやまちは繰りかえさないようにしましょう。 …といいたいところですが、あまり気にしない方がいいかもしれません。

    文字の打ち方を忘れてしまったときは、 [ヒント(H)] ボタンをクリックしてください。 ヒント欄に問題文の打ち方が表示されます。

    [ヒント(H)] ボタンを押すのにマウスを使うと、 手がキーボードから離れてしまうので、 かわりに、アクセスキーH を使うのがいいでしょう。 (Windows の場合は Alt+H、 Macintosh の場合は Ctrl+H。 Opera なら Shift+Esc H)

    [スタイル(Y)] でヒントの表示形式を選択できます。 [打鍵図] は、 tc2 や橋田表でも用いられている表示方法で、 特に T-Code を練習する場合におすすめです。 TUT-Code 文化圏の人は、[英字] がお好みかも知れません。 コンパクトに表記できてキーボードに依存しない [橋田ニモニック] も、試験的に導入しました。 なお、 「木を見て森を見る」形式の [ストローク表] は、 現在のバージョンでは機能しません。ごめんなさい。

  5. 入力をチェックする

    一行分の入力が終わったら、リターンキーを押してください。 入力したキーがチェックされ、結果が表示されます。 水色は正しく入力した文字、 赤色はまちがった文字です。

    次の問題文 (もしあれば) が表示されていますので、 続けて同じように入力します。

  6. 次のレッスンに進む/終了する

    レッスンの最後まで来ると、 『もう一度トライしますか?』と訊かれます。 同じレッスンをもう一度練習するときは a (Again) と入力して、 次のレッスンに進みたいなら n (Next) と入力して、リターンキーを押します。 また、q (Quit) と入力してリターンキーを押すと、 練習を終了し、総合成績を表示します。

    p (Previous) で前のレッスンに、 r (Random) でランダムなレッスンにジャンプします。 何も入力しないでただリターンキーを押した場合は、 n と同じです。

    キーのかわりに、[← 前(P)] [もう一度(A)] [次(N) →] [ランダム(R)] [終了 (Q)] の各ボタン (またはそのアクセスキー) を押してもかまいません。

    いったん終了したあとでも、 [もう一度(A)] ボタンなどをクリックすると、 練習を再開することができます。

漢索/JS

漢索/JS は、 「漢索板」 や、 「漢索窓」(漢直Win に付属しています) の JavaScript 版です。

使い方

  • 基本的な使い方

    漢索/JS のページに進み、 [検索(J)] 欄に、コードを調べたい文字 (列) を (IME や 漢直Win などを用いて) 入力します。 検索欄の下のコード欄に、打鍵図が表示されます。

  • リアルタイム検索

    デフォルトでは、入力とほぼ同時に自動で検索を行いますので、 [検索(M)] ボタンを押す必要はありません (通常の状態では、[検索(M)] ボタンは押せないはずです)。 うまく検索できない場合や、動作が非常に遅い場合は、 [設定(C)] セクションの [検索オプション] の [リアルタイム(R)] オプションのチェックをオフにしたうえで、 [検索(M)] ボタンを押して検索を行ってみてください。

  • 入力方式と表示形式

    [設定(C)] セクションで、 [入力方式(I)] (どのコード体系で調べるか) と [スタイル(Y)] (どのようにコードを表示するか) を選択できます。 入力方式もスタイルも、項目を複数選ぶことができます。 Windows の場合は Ctrl+クリックで、 MacOS の場合は Command+クリックで選択してください。

    ※ [入力方式(I)] の [ASCII] は、 「エスケープシーケンスのない JIS コード」あるいは、 「MSB (Most Significant Bit) を落とした EUC 日本語コード」で、 実は入力方式でもなんでもないのですが、面白そうなので入れてみました。 [スタイル(Y)] で [英字] を、 [キーボード(K)] で [QWERTY 日本語] を選択してお楽しみ下さい。

    ※ [スタイル(Y)] の [ストローク表] は、 「木を見て森を見る」形式の表示ですが、 現在のバージョンでは機能しません。ごめんなさい。

  • 応用例

    ブラウザの印刷機能を利用することで、 私家版の橋田表もどきを手軽に作ることができます。 [設定(C)] と [検索(L)] のタイトル部分をクリックすると、 不要な部分を非表示にできますので、この状態で印刷を行うとよいでしょう。 印刷のオプションやプリンタの設定で、必要に応じて、 縮尺や余白の大きさなどを設定してみてください。
